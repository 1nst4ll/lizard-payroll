## Prerequisites

*   A Render account.
*   Your project code pushed to a Git repository (GitHub, GitLab, or Bitbucket).
*   Familiarity with Render's dashboard and basic concepts.

## 1. Database Setup (PostgreSQL on Render)

Render offers managed PostgreSQL databases.

1.  **Create a New PostgreSQL Database:**
    *   Go to your Render Dashboard.
    *   Click "New" -> "PostgreSQL".
    *   Choose a name for your database (e.g., `payroll_db`), select a region, and a plan.
    *   Click "Create Database".
2.  **Note Database Credentials:** Once created, Render will provide you with the `External Database URL`, `Internal Database URL`, `Host`, `Port`, `User`, `Password`, and `Database` name. You will need the `External Database URL` for your backend service.

## 2. Backend Service Deployment (Laravel API)

The Laravel backend will be deployed as a Web Service on Render.

1.  **Create a New Web Service:**
    *   Go to your Render Dashboard.
    *   Click "New" -> "Web Service".
    *   Connect your Git repository where your backend code resides.
    *   Select the branch you want to deploy (e.g., `main` or `master`).
    *   Click "Next".
2.  **Configure Web Service:**
    *   **Name:** Choose a name for your backend service (e.g., `payroll-api`).
    *   **Region:** Select the same region as your PostgreSQL database for optimal performance.
    *   **Root Directory:** Specify the path to your backend project if it's not in the repository root (e.g., `backend/` or `server/`).
    *   **Runtime:** `PHP`
    *   **Build Command:** `composer install --no-dev --optimize-autoloader && php artisan migrate --force && php artisan key:generate && php artisan config:cache && npm install && npm run build`
    *   **Start Command:** `php artisan serve --host 0.0.0.0 --port $PORT`
    *   **Note:** The `php artisan migrate --force` command is included in the build command to ensure migrations are run automatically during deployment.
    *   **Plan:** Choose a suitable plan (e.g., Free, Starter, Pro).
3.  **Environment Variables:**
    *   Add the following environment variables under the "Environment" section:
        *   `APP_KEY`: A strong, random string (generated by `php artisan key:generate` locally).
        *   `DB_CONNECTION`: `pgsql`
        *   `DB_HOST`: Internal Database Host from your Render PostgreSQL database.
        *   `DB_PORT`: Internal Database Port from your Render PostgreSQL database.
        *   `DB_DATABASE`: `payroll_db`
        *   `DB_USERNAME`: `payroll_dev`
        *   `DB_PASSWORD`: Password from your Render PostgreSQL database.
        *   `APP_ENV`: `production`
        *   `APP_DEBUG`: `false`
        *   `VITE_API_URL`: The public URL of your deployed backend service (e.g., `https://payroll-api.onrender.com/api`). This will be used by your frontend.
    *   **Important:** Ensure these variables are marked as "Secret" if they contain sensitive information.
4.  **Health Check Path (Optional but Recommended):**
    *   If your Laravel app has a health check endpoint (e.g., `/health`), configure it under "Health Check Path" to ensure Render correctly monitors your service.
5.  **Create Web Service:** Click "Create Web Service". Render will now build and deploy your backend.

## 3. Frontend Service Deployment (Vue.js App)

The Vue.js frontend will be deployed as a Static Site on Render.

1.  **Create a New Static Site:**
    *   Go to your Render Dashboard.
    *   Click "New" -> "Static Site".
    *   Connect your Git repository where your frontend code resides.
    *   Select the branch you want to deploy.
    *   Click "Next".
2.  **Configure Static Site:**
    *   **Name:** Choose a name for your frontend service (e.g., `payroll-app-frontend`).
    *   **Region:** Select a region.
    *   **Root Directory:** Specify the path to your frontend project if it's not in the repository root (e.g., `frontend/` or `client/`).
    *   **Build Command:** `npm install && npm run build`
    *   **Publish Directory:** `dist` (This is the default output directory for Vite builds).
3.  **Environment Variables:**
    *   Add the following environment variable:
        *   `VITE_APP_API_URL`: The public URL of your deployed backend service (e.g., `https://payroll-api.onrender.com/api`). You can find this URL on your backend service's dashboard page on Render.
4.  **Create Static Site:** Click "Create Static Site". Render will now build and deploy your frontend.

## 4. Running Database Migrations on Render

After your backend service is deployed and connected to the Render PostgreSQL database, you need to run your database migrations to apply the schema to the production database.

You can do this via a "Shell" session on your backend service:

1.  Go to your backend service dashboard on Render.
2.  Click on the "Shell" tab.
3.  Run your migration command:
    ```bash
    php artisan migrate --force
    ```
    *   **Note:** The `--force` flag is necessary when running migrations in a production environment.

## 5. Custom Domains (Optional)

To use a custom domain (e.g., `app.yourpayroll.com`) instead of the default Render URL:

1.  Go to your service dashboard (backend or frontend).
2.  Click on the "Settings" tab.
3.  Scroll down to "Custom Domains" and click "Add a Custom Domain".
4.  Follow the instructions to add your domain and configure DNS records with your domain registrar.

## 6. Continuous Deployment

Render automatically deploys new changes whenever you push to the connected Git branch. This provides a seamless CI/CD pipeline.

## Troubleshooting Deployment Issues

*   **Build Failures:** Check the "Logs" tab on your Render service dashboard for build errors. Common issues include missing dependencies or incorrect build commands.
*   **Runtime Errors:** If your service deploys but doesn't run correctly, check the "Logs" tab for runtime errors.
*   **Environment Variables:** Ensure all necessary environment variables are correctly set on Render, especially database credentials and API URLs.
*   **Database Connection:** Verify that your backend service can connect to the PostgreSQL database. Check firewall rules if applicable (though Render usually handles this internally for services in the same region).

By following this guide, you should be able to successfully deploy your Payroll Web Application to Render.
