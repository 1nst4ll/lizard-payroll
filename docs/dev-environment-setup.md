## Prerequisites

Before you begin, ensure you have the following installed on your system:

*   **Operating System:** Windows 11 Pro (as per your local environment)
*   **Git:** For version control. Download from [git-scm.com](https://git-scm.com/).
*   **PHP:** Version 8.2 or later. Download from [php.net](https://www.php.net/downloads.php).
    *   Verify installation: `php -v`
*   **Composer:** PHP dependency manager. Download from [getcomposer.org](https://getcomposer.org/download/).
    *   Verify installation: `composer -v`
*   **Node.js:** Version 20.x or later (LTS recommended). Download from [nodejs.org](https://nodejs.org/).
    *   Verify installation: `node -v` and `npm -v`
*   **Laravel Installer:** Install globally: `composer global require laravel/installer`
*   **Laravel Sail (Optional, for Docker-based development):** Included with new Laravel projects.
*   **Vue CLI (Optional, for scaffolding):** Install globally: `npm install -g @vue/cli`
*   **PostgreSQL:** Database server. Download from [postgresql.org](https://www.postgresql.org/download/).
    *   You mentioned having XAMPP, psql, and pgAdmin4 installed, which should cover this.
*   **pgAdmin4:** PostgreSQL administration and development platform.
*   **VS Code:** Recommended IDE. Download from [code.visualstudio.com](https://code.visualstudio.com/).

## 1. Clone the Repository

First, clone the project repository to your local machine.

```bash
git clone [repository_url]
cd payroll-web-app # Or whatever your project's root directory is named
```

## 2. Backend Setup (PHP with Laravel)

Navigate to the backend project directory (`backend/`).

### 2.1 Project Initialization

If not already done, create the Laravel project with Jetstream (Inertia Stack):
```bash
laravel new backend --jet --stack=inertia --teams
```
Then navigate into the backend directory:
```bash
cd backend
```

### 2.2 Install Dependencies

```bash
composer install
npm install && npm run build
```
Additionally, install Laravel Socialite for Google OAuth and Spatie Permission for RBAC:
```bash
composer require laravel/socialite
composer require spatie/laravel-permission
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider" --tag="permission-migrations"
php artisan migrate
php artisan db:seed --class="PermissionSeeder" # You'll need to create this seeder
```
Note: For Laravel Socialite, you'll need to configure your `config/services.php` and `.env` with Google OAuth credentials. For Spatie Permission, you'll need to set up roles and permissions in your database seeders or migrations.

### 2.3 Environment Configuration

Create a `.env` file in the root of your backend project (`backend/.env`). This file will store sensitive information and configuration variables.

```
APP_NAME="Payroll Web App"
APP_ENV=local
APP_KEY=base64:YOUR_GENERATED_APP_KEY # This will be generated by `php artisan key:generate`
APP_DEBUG=true
APP_URL=http://localhost:8000

LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug

DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=payroll_db
DB_USERNAME=your_username
DB_PASSWORD=your_password

BROADCAST_DRIVER=log
CACHE_DRIVER=file
FILESYSTEM_DISK=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

MEMCACHED_HOST=127.0.0.1

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"

AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=
AWS_USE_PATH_STYLE_ENDPOINT=false

PUSHER_APP_ID=
PUSHER_APP_KEY=
PUSHER_APP_SECRET=
PUSHER_HOST=
PUSHER_PORT=443
PUSHER_SCHEME=https
PUSHER_APP_CLUSTER=mt1

VITE_APP_NAME="${APP_NAME}"
VITE_API_URL=http://localhost:8000/api # Adjust if your backend runs on a different port or path
```
Run `php artisan key:generate` to generate your `APP_KEY`.

### 2.4 Database Setup

Ensure your PostgreSQL server is running.

1.  **Create Database:** If you haven't already, create the database specified in your `.env` file (e.g., `payroll_db`). You can do this via `pgAdmin4` or the `psql` command line:
    ```bash
    psql -U your_username -c "CREATE DATABASE payroll_db;"
    ```
2.  **Run Migrations:** Once the database is created, run the database migrations to set up the schema.

    ```bash
    php artisan migrate
    ```

### 2.5 Run the Backend

```bash
php artisan serve
```

This will start the Laravel backend server, typically on `http://localhost:8000`.

## 3. Frontend Setup (Vue.js with Vite & TypeScript)

Navigate to the frontend project directory (`frontend/`).

### 3.1 Project Initialization (Integrated with Laravel Jetstream)

With Laravel Jetstream (Inertia Stack), the frontend (Vue.js) is typically integrated directly within the Laravel project. You will find the Vue.js components in `resources/js/`.

If you need a separate frontend project for any reason, you would create it as follows:
```bash
npm create vite@latest frontend -- --template vue-ts
```
Then navigate into the frontend directory:
```bash
cd frontend
```

### 3.2 Install Dependencies (for integrated frontend)

Navigate to the backend project directory (`backend/`).
```bash
npm install
```
This will install the necessary frontend dependencies for Jetstream.

Additionally, install Tailwind CSS, PostCSS, Autoprefixer, and `shadcn/ui` related packages if not already included by Jetstream:
```bash
npm install -D tailwindcss postcss autoprefixer
npm install tailwindcss-animate clsx tailwind-merge class-variance-authority @radix-vue/react-slot
```
Note: `shadcn/ui` for Vue is still under development, so some components might need manual setup or alternatives.

### 3.3 Environment Configuration (for integrated frontend)

The frontend will use the `VITE_APP_API_URL` defined in the backend's `.env` file.

### 3.4 Run the Frontend (Integrated with Laravel)

Navigate to the backend project directory (`backend/`).
```bash
npm run dev
```
This will compile your frontend assets using Vite and serve them.

This will start the Vite development server, typically on `http://localhost:5173`, and open the application in your default web browser.

## Troubleshooting

*   **Port Conflicts:** If a port is already in use, you might see an error. You can either change the port in your `.env` file or find and terminate the process using that port.
*   **Database Connection Issues:** Double-check your `DATABASE_URL` in the backend `.env` file. Ensure PostgreSQL is running and accessible.
*   **Missing Dependencies:** If you encounter errors about missing modules, try running `npm install` again in the respective project directory.

By following these steps, you should have your local development environment for the Payroll Web Application up and running.
